# Sakura 做题集软著材料包

本目录用于整理 `Sakura 做题集智能错题复盘系统 V1.0` 的计算机软件著作权登记材料草稿。

## 已准备材料

- `01_软著申请信息草稿.md`：用于填写登记系统时参考，包含软件名称、版本、开发方式、功能概述、技术特点和待补信息。
- `02_软件设计说明书.md`：偏技术说明，适合提交为软件文档材料。
- `03_用户操作手册.md`：偏用户说明，适合提交为软件文档材料，也适合后续转 Word。
- `04_源代码鉴别材料_分页版.txt`：按前 30 页、后 30 页思路生成的源代码材料，带页码分隔，便于打印检查。
- `04_源代码鉴别材料_连续代码.txt`：同一份代码的连续文本版本，便于复制、排版或重新分页。
- `generate_source_material.py`：从项目源码重新生成源代码鉴别材料的脚本。

## 你需要补充的信息

- 著作权人真实姓名或单位名称。
- 身份证号或统一社会信用代码。
- 联系地址、电话、邮箱。
- 开发完成日期。
- 首次发表状态和首次发表日期。
- 是否有合作开发、委托开发、职务开发等情况。

## 建议软件名称

首选：

```text
Sakura 做题集智能错题复盘系统 V1.0
```

备选：

```text
Sakura 理工科学习智能错题管理与复习系统 V1.0
```

## 重要提醒

- 不要把 `.env`、API Key、服务器密码、真实身份证号、手机号、邮箱、上传 PDF、SQLite 数据库提交到 GitHub。
- 源代码鉴别材料只应使用项目源码，不应包含用户上传的教材、题库、PDF 或个人学习数据。
- 如果你已经把 GitHub 仓库公开、或者已经通过公网域名给他人访问，首次发表状态和日期需要按真实情况填写。
- 如果只是本地开发自用、没有正式对外发布，可以按实际情况选择“未发表”。

## 后续提交前建议

1. 将 `03_用户操作手册.md` 或 `02_软件设计说明书.md` 转成 Word。
2. 检查文档里的所有 `[待填写]` 是否已经替换。
3. 检查源代码材料是否没有敏感信息。
4. 根据登记平台要求上传或打印材料。
