# Sakura 做题集智能错题复盘系统 V1.0 软件设计说明书

## 1. 软件概述

Sakura 做题集智能错题复盘系统是一套本地优先的学习管理与 AI 辅助复盘软件。系统面向高等数学、计算机专业课程和其他理工科学习场景，解决传统纸质题本和 PDF 题本难以持续复盘、错题难以按知识点归类、学习记录难以形成长期反馈的问题。

系统将 PDF 题本导入、题目拆分、题图保存、章节识别、错题标注、错因分析、每日练习、间隔复习、教材精读、AI 学习教练、推送提醒和数据迁移组织在同一套软件中。学习者可以把已做题目沉淀为可检索、可统计、可复习、可被 AI 分析的学习数据。

## 2. 设计目标

1. 建立统一的题库与错题管理入口，使不同资料、不同科目、不同章节的题目可被集中管理。
2. 减少对大模型的无效调用，PDF 拆页、题图保存、章节清洗、统计计算和复习队列优先在本地完成。
3. 通过错因标签、多刷记录和复习阶段记录，保留学习者每次复盘的真实证据。
4. 让 AI 学习教练基于题目、错因、复习记录、老师记忆和学习档案给出个性化建议。
5. 支持本地运行和轻量云端部署，便于个人自用、演示和数据迁移。

## 3. 总体架构

系统采用前后端分离但轻量化集成的架构：

- 前端：原生 HTML、CSS、JavaScript 模块化实现，所有页面由 `static/index.html` 承载，各功能模块拆分到 `static/js` 子目录中。
- 后端：Python HTTP 服务入口为 `app.py`，路由、数据库、鉴权、内容处理、复习、AI 和系统配置拆分到 `sakura` 包内。
- 数据层：SQLite 保存题库、资料、做题状态、错因记录、老师记忆、反思归档、提醒配置和安全事件。
- 文件层：上传的 PDF、题图、导出文件和备份文件保存在本地数据目录中。
- AI 层：通过 OpenAI Compatible API 调用 DeepSeek 或其他兼容模型。

## 4. 后端模块设计

### 4.1 core 基础模块

`sakura/core` 负责系统基础设施：

- `config.py`：读取环境变量和运行时配置。
- `db.py`：初始化 SQLite 表结构，提供数据库连接。
- `routes.py`：统一管理 HTTP 路由。
- `http.py`：封装 JSON 响应、静态资源响应和错误响应。
- `auth.py`：处理登录保护和会话签名。
- `security.py`：处理访问密码规则、登录失败限速、分级锁定和安全事件。
- `parse.py`：解析请求体、查询参数和表单数据。

### 4.2 content 内容模块

`sakura/content` 负责题库、资料和教材：

- `pdf.py`：PDF 页面渲染、文本提取、题图生成。
- `importer.py`：处理资料导入流程。
- `classify.py`：清洗章节文本、知识点和资料分类。
- `documents.py`：资料和做题本管理。
- `questions.py`：题目查询、详情、状态更新和多刷复盘记录。
- `filters.py`：科目、资料、章节和知识点筛选。
- `textbook.py`：教材页和段落处理。
- `models.py`：题目和资料的数据结构转换。

### 4.3 review 复习模块

`sakura/review` 负责错题复盘和练习：

- `daily.py`：生成每日练习、处理回填状态、插入复习记录。
- `retention.py`：维护复习阶段和间隔复习逻辑。
- `hints.py`：生成启发式提示。
- `insights.py`：整理错题洞察和统计。
- `reflection.py`：生成周、月复盘归档。
- `export.py`：按条件导出错题 PDF。

### 4.4 ai 智能教练模块

`sakura/ai` 负责大模型调用与学习档案：

- `client.py`：封装 OpenAI Compatible API 请求。
- `coach.py`：生成学习教练提示词和对话上下文。
- `profile.py`：维护学习档案、薄弱点和计划建议。
- `teacher_memory.py`：管理老师记忆、学科分类、搜索和压缩模板。

### 4.5 system 系统模块

`sakura/system` 负责系统级配置：

- `notifications.py`：企业微信、PushPlus、邮箱推送。
- `reminders.py`：早晚提醒、天气提醒和打卡入口。
- `weather.py`：天气查询配置。
- `email.py`：SMTP 邮箱发送。
- `backup.py`：数据备份、恢复和迁移。
- `settings.py`：运行时配置读写。
- `migration.py`：迁移任务管理。

## 5. 前端模块设计

前端通过 `static/index.html` 提供页面骨架，通过 `static/styles.css` 统一视觉风格，通过 `static/js` 内的模块进行功能绑定。

主要前端模块包括：

- `static/js/core/app.js`：初始化应用、导航切换、公共状态和事件绑定。
- `static/js/content/dashboard.js`：总览、知识点分布和统计卡片。
- `static/js/content/library.js`：题库筛选和题目列表。
- `static/js/content/documents.js`：资料、做题本和模拟卷管理。
- `static/js/content/question_detail.js`：题目详情、标注、AI 解析和复盘记录。
- `static/js/content/textbook.js`：教材精读和教材问答。
- `static/js/review/daily.js`：每日练习。
- `static/js/review/mistakes.js`：错题本。
- `static/js/review/reflection.js`：总结与反思。
- `static/js/review/mistake_export.js`：错题导出。
- `static/js/ai/ai_chat.js`：AI 老师对话、记忆导入和记忆检索。
- `static/js/system/reminders.js`：提醒、天气、推送和安全设置。
- `static/js/system/backup.js`：数据备份和迁移。

## 6. 数据库设计

系统采用 SQLite 保存核心数据。主要数据表包括：

- 资料表：保存做题本、模拟卷、教材的名称、科目、类型和创建时间。
- 题目表：保存题目所属资料、页码、章节、知识点、题图路径、文本摘要和题目状态。
- 复习记录表：保存做题日期、状态、错因、备注、复习阶段和下次复习时间。
- 多刷记录表：保存每次复盘的日期、状态、错因和补充说明。
- 学习档案表：保存学习目标、考试日期、每日学习时长和薄弱点。
- 老师记忆表：保存 AI 老师长期记忆、学科分类、来源和摘要。
- 反思归档表：保存周、月复盘结果。
- 提醒配置表：保存推送通道、提醒时间、天气城市和打卡入口。
- 安全事件表：保存登录失败、锁定、登录成功和密码更新记录。

## 7. 核心业务流程

### 7.1 PDF 做题本导入流程

1. 用户选择科目、资料名称、资料类型和页码范围。
2. 系统读取 PDF 页面并进行渲染。
3. 系统保存题图和页面文本。
4. 系统从页面右上角或文本区域提取章节信息。
5. 系统清洗重复章节、公众号、水印和无关字样。
6. 系统将题目写入题库。
7. 用户可在题库中按科目、资料、章节和知识点筛选。

### 7.2 错题标注流程

1. 用户打开题目详情。
2. 用户选择做题状态。
3. 如果标记为错题或半会题，用户选择错因标签。
4. 用户可填写本次复盘补充说明。
5. 系统保存状态，并追加多刷复盘记录。
6. 系统更新复习阶段和下次复习时间。

### 7.3 每日练习流程

1. 系统读取错题、半会题、到期复习题和历史薄弱题。
2. 系统按科目、资料、章节和自定义推送规则筛选候选题。
3. 系统根据题量设置生成练习列表。
4. 用户完成题目后回填状态、错因和备注。
5. 系统更新复习计划和统计结果。

### 7.4 AI 分析流程

1. 用户在题目详情中请求提示、解析或变式。
2. 系统读取题目文本、题图上下文、错因标签和复盘记录。
3. 系统生成提示词并调用大模型 API。
4. 前端使用 MathJax 渲染 AI 返回的 LaTeX 公式。
5. 用户可将重要解释导入老师记忆。

### 7.5 教材精读流程

1. 用户上传教材 PDF。
2. 系统按页保存教材截图和段落文本。
3. 用户指定页码、段落或自然语言问题。
4. 系统检索相关教材内容并发送给 AI。
5. AI 返回解释、例子或学习建议。
6. 用户可将高价值解释压缩为老师记忆。

## 8. 安全与数据保护

系统默认不将用户上传的 PDF、题图、数据库、API Key 或服务器密钥提交到 Git 仓库。运行时配置通过 `.env` 管理，公开仓库仅保留 `.env.example` 示例。

公网部署时，系统支持管理员访问密码。密码需满足复杂度要求：至少 12 位，包含字母、数字和特殊字符，不包含空格或换行。系统对同一来源 IP 的连续失败登录进行限速，并按 5 分钟、10 分钟、1 小时、1 年逐级锁定。首次触发锁定时，系统可通过配置的推送通道发送安全提醒。

## 9. 外部接口设计

系统主要调用以下外部服务：

- OpenAI Compatible API：用于错题解析、提示、变式、周期反思、教材问答和 AI 老师对话。
- 企业微信机器人 Webhook：用于每日提醒、打卡入口和安全告警。
- PushPlus：作为微信推送备用通道。
- SMTP 邮箱：作为邮件推送备用通道。
- 天气查询接口：用于天气提醒。

系统设计上将本地功能和 AI 功能分离。PDF 导入、题库筛选、状态记录、统计图表和复习调度不依赖 AI；只有需要语言解释、计划建议或总结归纳时才调用大模型。

## 10. 运行部署

本地运行步骤：

1. 安装 Python 依赖。
2. 复制 `.env.example` 为 `.env`。
3. 根据需要填写 API Key、推送通道和登录密码。
4. 执行 `python app.py`。
5. 浏览器访问 `http://127.0.0.1:8000`。

服务器部署步骤：

1. 将项目代码上传到服务器。
2. 安装 Python 和依赖。
3. 配置 `.env`。
4. 使用 systemd 或其他进程管理工具运行服务。
5. 配置 Nginx 反向代理和访问限制。
6. 如使用域名，按服务商要求配置 DNS 和 HTTPS。

## 11. 设计创新点

1. 以错题证据链为核心，而不是单纯聊天式 AI 工具。
2. 将 PDF 题本、模拟卷、教材精读和 AI 老师放在同一学习闭环中。
3. 通过错因标签和多刷记录保留元认知信息。
4. 通过老师记忆和学科分类实现个性化学习上下文。
5. 通过本地优先策略降低 token 消耗和隐私泄露风险。
6. 通过推送和打卡机制把复习任务转化为日常行动。

## 12. 版本说明

V1.0 版本已实现题本导入、题库管理、错题标注、每日练习、AI 解析、教材精读、老师记忆、反思归档、推送提醒、数据迁移和访问安全等核心功能，可作为个人学习原型系统运行。
