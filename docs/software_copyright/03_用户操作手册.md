# Sakura 做题集智能错题复盘系统 V1.0 用户操作手册

## 1. 手册说明

本手册用于说明 Sakura 做题集智能错题复盘系统 V1.0 的安装、配置、使用流程和常见问题。软件面向需要长期刷题、错题复盘和 AI 辅助学习的个人用户，尤其适合高等数学、计算机专业课、模拟卷训练和教材精读场景。

## 2. 软件简介

Sakura 做题集是一套本地优先的学习工作台。用户可以上传 PDF 做题本、模拟卷或教材，系统会将页面拆分为题目或教材页，保存题图和文本，并按照科目、资料、章节和知识点进行管理。用户在做题后可以标记题目状态、错因和复盘补充，系统会生成每日练习、统计图表和周期反思。

当用户需要进一步讲解时，系统可以调用 AI 接口，生成启发式提示、完整解析、举一反三变式、教材段落解释和个性化学习规划。系统还支持企业微信、PushPlus 和邮箱提醒，帮助用户按时复习和打卡。

## 3. 运行准备

### 3.1 环境要求

建议运行环境：

- Python 3.10 或以上版本。
- Windows、Linux 或轻量云服务器。
- Chrome、Edge 或其他现代浏览器。
- 足够的本地磁盘空间，用于保存 PDF、题图和数据库。

### 3.2 安装依赖

在项目目录中执行：

```powershell
pip install -r requirements.txt
```

### 3.3 配置环境变量

复制 `.env.example` 为 `.env`：

```powershell
copy .env.example .env
```

如仅使用本地题库和错题管理，可以不填写 AI Key。  
如需要 AI 解析、AI 老师、教材问答和反思总结，需要填写大模型 API 配置。

常用配置包括：

- `PORT`：服务端口。
- `APP_PUBLIC_URL`：公网访问地址，用于生成推送链接。
- `SAKURA_ADMIN_PASSWORD`：公网部署时的管理员登录密码。
- `SAKURA_AUTH_SECRET`：登录签名密钥。
- `LLM_API_KEY`：大模型 API Key。
- `LLM_BASE_URL`：OpenAI Compatible API 地址。
- `LLM_MODEL`：模型名称。
- `WEWORK_BOT_WEBHOOK`：企业微信机器人 Webhook。
- `PUSHPLUS_TOKEN`：PushPlus Token。
- `EMAIL_ENABLED`：是否启用邮箱推送。

### 3.4 启动系统

执行：

```powershell
python app.py
```

启动成功后，在浏览器中打开：

```text
http://127.0.0.1:8000
```

如果配置了公网服务器，则打开对应的公网地址。

## 4. 登录与安全

如果配置了管理员密码，首次访问系统会进入登录页。输入密码后即可进入学习面板。

密码建议满足以下要求：

- 至少 12 位。
- 包含字母、数字和特殊字符。
- 不使用生日、姓名、手机号等容易猜测的信息。

如果连续多次输入错误，系统会触发限速和锁定。锁定事件会被记录，必要时会通过已配置的推送通道发送提醒。

## 5. 总览面板

进入系统后，用户首先看到总览页面。总览页面展示题库数量、错题数量、待复习数量、近期练习情况和知识点分布。

用户可通过总览页面了解：

- 当前已经导入多少题。
- 当前有多少错题和待复习题。
- 最近一段时间的做题正确率。
- 哪些章节或知识点薄弱。
- 哪些错因出现频率较高。

知识点分布默认不展示全部内容，用户需要先选择科目和资料，再查看对应章节和知识点。

## 6. 上传做题本

### 6.1 新增资料

在做题本或模拟卷管理区域，用户可以选择上传 PDF。上传时需要填写：

- 科目。
- 资料或做题本名称。
- 类型，例如普通做题本、模拟卷或教材。
- 起始页和结束页。

系统支持只导入指定页码范围，避免重复上传整本资料。

### 6.2 章节识别

系统会尝试从 PDF 页面右上角或页面文本中识别章节内容。识别结果会进行清洗，去除重复章节名、水印、公众号、基础篇等干扰信息。

如果章节识别不准确，用户可以在题目详情或资料编辑中手动修正。

### 6.3 资料编辑

如果上传时资料名称、科目或类型填写错误，可以在资料管理页面编辑，不需要删除后重新上传。

### 6.4 删除资料

当某个做题本过旧或需要重新导入时，可以删除资料。删除资料时，系统会同步处理其关联题目和复习记录，避免题库中残留无效记录。

## 7. 题库筛选

题库页面支持按以下条件筛选：

- 科目。
- 资料或做题本。
- 章节。
- 知识点。
- 做题状态。
- 关键词。

筛选逻辑遵循科目和资料的层级关系。用户未选择科目和资料时，章节和知识点不会盲目展示全部内容；选择后，系统只展示与当前科目和资料对应的章节、知识点。

## 8. 题目详情

打开题目详情后，用户可以查看题图、页码、科目、资料、章节、知识点和当前状态。

题图支持放大查看。用户可以双击题图或点击放大按钮查看细节，适合处理公式较多或图片较小的题目。

题目详情中可以进行以下操作：

- 标记做对。
- 标记做错。
- 标记半会。
- 标记需复习。
- 修改章节和知识点。
- 填写本次复盘补充。
- 生成 AI 提示。
- 生成完整解析。
- 生成举一反三变式。

## 9. 错因标签

当用户标记题目为做错、半会或需复习时，建议选择错因标签。系统默认提供以下错因：

- 计算失误。
- 公式遗忘。
- 逻辑死角。
- 题意理解偏差。

错因标签不是为了责备用户，而是为了帮助系统识别学习习惯。例如，经常出现计算失误，说明需要增加草稿规范和算后检查；经常出现公式遗忘，说明需要加强公式卡片和基础复习；经常出现逻辑死角，说明需要补前置知识。

## 10. 多刷复盘记录

传统错题本常见问题是：第二次做题时会覆盖第一次备注，导致学习过程丢失。Sakura 做题集采用多刷复盘记录。

每次用户在题目详情或每日练习中填写复盘补充，系统都会按日期追加一条独立记录。记录内容包括：

- 日期。
- 做题状态。
- 错因标签。
- 复盘补充。
- 复习阶段。

这样用户可以看到同一道题从第一次错误到后续逐步掌握的过程。

## 11. 每日练习

每日练习不是从原始题库随机抽题，而是根据学习记录生成。系统优先考虑：

- 已标记错题。
- 半会题。
- 到期复习题。
- 正确率较低章节。
- 用户自定义推送规则。

用户可以按科目、资料、章节和题量设置练习范围。例如，用户可以设置 408 科目下的机组资料每天推送 5 道错题，而不是每个小节都推送 5 道。

完成练习后，用户可回填状态和补充说明。系统会据此更新复习阶段和下次复习时间。

## 12. 间隔复习

系统参考 1、3、7、14、30 天的节奏维护复习阶段。当题目做错时，会进入复习队列；后续做对时，复习间隔会逐步拉长；如果再次做错，系统会重新降低掌握程度。

这种方式的目的不是让错题永远留在错题本里，而是让错题在合适时间重新出现，直到用户真正掌握。

## 13. AI 启发式提示

题目详情中可以使用 `Get Hint` 或类似按钮获得提示。提示分为三级：

- Level 1：只给核心概念、定理或方向。
- Level 2：给第一步关键变形、辅助线或切入点。
- Level 3：生成完整解析。

这种设计避免用户一开始就看到完整答案，有利于保留思考过程。

## 14. AI 完整解析

当用户需要详细讲解时，可以生成完整解析。系统会将题目文本、题图信息、章节、错因和复盘记录整理后发送给 AI。AI 返回后，前端会使用 MathJax 渲染数学公式。

如果公式显示异常，用户可以刷新页面或重新生成；系统会尽量规范 Markdown 和 LaTeX 的显示。

## 15. 举一反三变式

系统支持三类变式训练：

- Base：换数不换逻辑，帮助用户确认基本方法是否掌握。
- Advanced：改变求解目标，训练逆向迁移能力。
- Pro：跨章节综合，训练综合应用能力。

举一反三适合在错题复盘后使用，目的是确认用户是否真的理解了题型迁移规律。

## 16. AI 老师对话

AI 老师面板用于学习规划、错题复盘、知识点讲解和 API 测试。用户可以直接提问，也可以让 AI 根据学习档案和老师记忆生成建议。

AI 老师不会默认知道用户所有学习情况，只有在用户导入记忆、更新学习档案或关联错题记录后，回复才会更加个性化。

## 17. 老师记忆

用户可以将重要对话、教材解释或复盘总结导入老师记忆。导入时可以选择或创建学科，例如高等数学、408、机组、操作系统等。

老师记忆支持：

- 按学科分类。
- 关键词搜索。
- 查看历史记忆。
- 智能压缩。
- 自定义压缩规则。

记忆压缩类似于把长对话压缩为可长期保留的学习档案，避免 AI 老师每次都从零开始了解用户。

## 18. 教材精读

教材精读模块用于处理用户上传的教材 PDF。系统会按页保存教材截图和段落文本。用户可以向 AI 提问：

- 第 151 页第 3 段是什么意思。
- 这一段和前面的定理有什么关系。
- 能否用更简单的话解释。
- 能否举一个考试题例子。

如果用户认为某次解释很有价值，可以将其压缩导入老师记忆。

## 19. 总结与反思

总结与反思模块按周和月生成学习复盘。系统会按科目统计：

- 本周期做题数。
- 做对数。
- 做错数。
- 正确率。
- 薄弱章节。
- 高频错因。
- 待复习题。

用户可以调用 AI 生成反思报告。AI 报告应指出当前不足、阶段重点和后续规划。

## 20. 错题导出

错题导出功能用于将错题整理成 PDF，方便打印。用户可以按以下条件筛选：

- 科目。
- 资料或做题本。
- 章节。
- 知识点。
- 做题状态。
- 自定义选择。

系统支持全选和手动选择。导出的 PDF 会尽量保留原始题图，方便线下复盘。

## 21. 模拟卷管理

模拟卷和普通做题本同级管理，拥有独立上传入口。模拟卷通常没有固定章节或右上角章节标识，系统会按页保存题图，并允许用户后续手动归类。

如果一页中包含多道题，用户可以通过题目边界微调或后续手动拆分进行处理。

## 22. 提醒与打卡

系统支持早晚提醒、天气提醒和打卡入口。用户可以设置：

- 早间提醒是否开启。
- 早间提醒时间。
- 晚间提醒是否开启。
- 晚间提醒时间。
- 天气提醒是否开启。
- 天气提醒时间。
- 打卡入口类型。

推送通道可以选择企业微信机器人、PushPlus 或邮箱。企业微信机器人是当前推荐通道。

## 23. 推送通道配置

### 23.1 企业微信机器人

用户在企业微信群中添加机器人，复制 Webhook 地址，填入系统配置并保存。点击测试推送后，如果企业微信群收到消息，说明配置成功。

### 23.2 PushPlus

用户填写 PushPlus Token 后可以测试推送。如果出现实名认证相关错误，通常表示 PushPlus 账户侧未完成认证，并非 Sakura 配置丢失。

### 23.3 邮箱推送

用户可以填写 SMTP Host、端口、加密方式、发件邮箱、授权码、收件人和显示名称。QQ 邮箱需要使用授权码，不是登录密码。

## 24. 数据备份与迁移

系统支持本地数据导出和恢复。迁移范围包括：

- 题库记录。
- 错题状态。
- 多刷复盘记录。
- 学习档案。
- 老师记忆。
- 资料元数据。
- 必要的本地文件索引。

迁移时不应公开 `.env`、API Key、服务器密钥、数据库文件和上传 PDF。

## 25. 常见问题

### 25.1 PDF 识别不准怎么办

可以重新上传指定页码，也可以在题目详情中手动编辑章节和知识点。对于没有章节标识的模拟卷，建议后续人工归类。

### 25.2 AI 没有配置 Key 能用吗

本地题库、错题管理、统计、复习队列和导出功能可以使用。AI 解析、AI 老师、教材问答和反思总结需要配置大模型 API。

### 25.3 为什么错题做对后没有立刻消失

系统不是简单删除错题，而是更新复习阶段。题目做对后会逐步拉长复习间隔，直到稳定掌握。

### 25.4 为什么推送测试成功但定时提醒没有收到

需要检查提醒是否开启、时间是否正确、服务器时区是否正确、后台定时任务是否运行，以及推送通道是否仍然可用。

### 25.5 如何避免数据丢失

建议定期导出备份。重要修改前，先备份数据库和上传文件目录。

## 26. 注意事项

1. 用户应自行确认上传 PDF、教材和题库的版权合规性。
2. 不要把个人 API Key、Webhook、服务器密码写入公开仓库。
3. 公网部署时必须设置访问密码。
4. 演示给朋友使用时建议开启 demo 模式，避免他人修改数据。
5. 软著或比赛材料中不要包含真实学习隐私数据。

## 27. 结束语

Sakura 做题集的目标是让做过的题不浪费，让错过的知识点不遗漏。系统通过题库沉淀、错因标注、间隔复习、AI 讲解和长期记忆，把零散刷题过程整理为可持续复盘的学习闭环。
