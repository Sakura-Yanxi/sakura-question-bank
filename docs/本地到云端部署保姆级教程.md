# Sakura 做题集：从 Python 到云服务器的保姆级部署教程

这份教程写给第一次接触 Python 项目部署的同学。你不需要提前懂后端、Nginx、Linux 或云服务器，只要按步骤做，就能把本地可运行的 Sakura 做题集部署到一台 Ubuntu 云服务器上，让电脑和手机都能通过公网访问。

教程目标：

1. 在本地安装 Python 并跑通 Sakura 做题集。
2. 理解哪些是代码、哪些是数据、哪些是密钥。
3. 在云服务器安装运行环境。
4. 用 systemd 让项目后台运行并自动重启。
5. 用 Nginx 把公网 80 / 443 端口转发到 Sakura。
6. 配置域名、HTTPS、登录保护、AI Key 和微信提醒。
7. 把本地数据迁移到云端。

> 示例系统以 Ubuntu 22.04 / 24.04 为主。阿里云、腾讯云、Azure、轻量云服务器、普通 VPS 都可以参考。命令里出现的 `你的服务器公网IP`、`你的域名`、`你的用户名` 都需要替换成自己的。

---

## 0. 先理解整体结构

Sakura 做题集可以分成三类东西：

```text
代码：app.py、sakura_*.py、static/、requirements.txt
数据：data/ 目录里的数据库、上传 PDF、拆分题图、教材页图
密钥：.env 里的 API Key、登录密码、企业微信 Webhook、PushPlus Token
```

部署时要记住一条原则：

```text
代码可以上传 GitHub。
数据用备份迁移。
密钥只写在服务器 .env，不要提交 GitHub。
```

最终云服务器上的推荐结构是：

```text
/opt/sakura-study/
├── app.py
├── notify_daily.py
├── requirements.txt
├── .env
├── static/
├── sakura_*.py
└── data/
    ├── gaoshu_demo.sqlite3
    ├── uploads/
    └── pages/
```

访问链路是：

```text
浏览器 / 手机
  -> http://你的域名 或 http://服务器IP
  -> Nginx 监听 80 / 443
  -> 转发到 127.0.0.1:8000
  -> Python app.py 提供 Sakura 服务
```

## 1. 本地先跑通项目

这一部分在你的 Windows 电脑上做。

### 1.1 安装 Python

推荐安装 Python 3.11 或 3.12。

下载地址：

```text
https://www.python.org/downloads/
```

Windows 安装时一定要勾选：

```text
Add Python to PATH
```

安装完成后打开 PowerShell，输入：

```powershell
python --version
pip --version
```

能看到版本号，例如：

```text
Python 3.12.x
pip 24.x
```

就说明 Python 安装成功。

### 1.2 进入项目目录

假设项目在桌面：

```powershell
cd C:\Users\你的用户名\Desktop\demo
```

如果不知道路径，可以在文件夹空白处按住 `Shift` 右键，选择“在终端中打开”。

### 1.3 创建虚拟环境

虚拟环境可以把这个项目的依赖和电脑里其他 Python 项目隔离开。

```powershell
python -m venv .venv
```

激活虚拟环境：

```powershell
.\.venv\Scripts\Activate.ps1
```

如果 PowerShell 提示不允许运行脚本，执行：

```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

然后重新激活：

```powershell
.\.venv\Scripts\Activate.ps1
```

看到命令行前面出现 `(.venv)`，说明激活成功。

### 1.4 安装依赖

```powershell
pip install --upgrade pip
pip install -r requirements.txt
```

如果下载很慢，可以换清华源：

```powershell
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### 1.5 配置本地环境变量

复制示例配置：

```powershell
copy .env.example .env
```

用记事本或 VS Code 打开 `.env`，先保留最基础配置即可：

```env
PORT=8000
APP_PUBLIC_URL=http://127.0.0.1:8000
SAKURA_DEMO_MODE=0
```

如果你要测试 AI，再填写：

```env
LLM_API_KEY=你的APIKey
LLM_BASE_URL=https://api.deepseek.com/v1
LLM_MODEL=deepseek-chat
```

如果暂时没有 AI Key，可以先不填。PDF 导入、题库筛选、错题标注、统计和本地复习仍然能用。

### 1.6 启动本地服务

```powershell
python app.py
```

浏览器打开：

```text
http://127.0.0.1:8000
```

能看到 Sakura 做题集页面，说明本地已经跑通。

停止服务的方法：

```text
Ctrl + C
```

## 2. 准备云服务器

这一部分在云服务商控制台做。

### 2.1 推荐配置

只给几个朋友体验的小 demo：

```text
CPU：1 核或 2 核
内存：1 GB 到 2 GB
硬盘：20 GB 到 40 GB
系统：Ubuntu 22.04 或 Ubuntu 24.04
```

长期自用并上传较多 PDF：

```text
CPU：2 核
内存：2 GB 或以上
硬盘：40 GB 或以上
系统：Ubuntu 22.04 或 Ubuntu 24.04
```

注意：PDF 拆分后会产生题图，`data/` 会逐渐变大。上传大量做题本时，硬盘比 CPU 更容易先不够。

### 2.2 开放安全组 / 防火墙端口

在云服务商控制台里找到“安全组”或“防火墙”，开放：

```text
22   SSH 登录服务器
80   HTTP 网页访问
443  HTTPS 网页访问
```

不要直接开放 `8000`。Sakura 的 Python 服务只让服务器内部访问，公网交给 Nginx。

### 2.3 用 SSH 登录服务器

Windows PowerShell 输入：

```powershell
ssh admin@你的服务器公网IP
```

如果你的服务器用户名是 `root`：

```powershell
ssh root@你的服务器公网IP
```

如果云服务商给了特殊端口，例如 `49438`：

```powershell
ssh root@你的服务器公网IP -p 49438
```

第一次连接会问：

```text
Are you sure you want to continue connecting?
```

输入：

```text
yes
```

然后输入密码，能看到服务器命令行就登录成功。

## 3. 服务器安装基础环境

以下命令都在 SSH 登录后的服务器终端里执行。

### 3.1 更新系统

```bash
sudo apt update
sudo apt upgrade -y
```

### 3.2 安装必备软件

```bash
sudo apt install -y python3 python3-venv python3-pip nginx git unzip curl fail2ban
```

检查版本：

```bash
python3 --version
nginx -v
git --version
```

### 3.3 给小内存服务器加 swap

如果服务器只有 1 GB 或 2 GB 内存，建议加 2 GB swap，避免上传大 PDF 或恢复备份时卡死。

```bash
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
free -h
```

看到 `Swap` 有容量就成功。

## 7. 用 systemd 让项目后台运行

systemd 可以做到：

- 服务器重启后 Sakura 自动启动。
- 程序崩溃后自动重启。
- 用统一命令查看日志。

创建服务文件：

```bash
sudo nano /etc/systemd/system/sakura-study.service
```

写入：

```ini
[Unit]
Description=Sakura Study App
After=network.target

[Service]
Type=simple
User=admin
WorkingDirectory=/opt/sakura-study
Environment=PYTHONUNBUFFERED=1
ExecStart=/opt/sakura-study/.venv/bin/python /opt/sakura-study/app.py
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

如果你的服务器用户名不是 `admin`，先查看当前用户名：

```bash
whoami
```

然后把服务文件里的：

```ini
User=admin
```

改成你的用户名，例如：

```ini
User=root
```

启动服务：

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now sakura-study
sudo systemctl status sakura-study
```

看到：

```text
active (running)
```

说明启动成功。

查看实时日志：

```bash
sudo journalctl -u sakura-study -f
```

退出日志：

```text
Ctrl + C
```

常用命令：

```bash
sudo systemctl restart sakura-study
sudo systemctl stop sakura-study
sudo systemctl start sakura-study
sudo systemctl status sakura-study
```

## 8. 配置 Nginx 反向代理

Nginx 负责接收公网访问，再转发到 Sakura 的本地 8000 端口。

创建配置：

```bash
sudo nano /etc/nginx/sites-available/sakura-study
```

如果暂时没有域名，写：

```nginx
server {
    listen 80;
    server_name _;

    client_max_body_size 2G;
    proxy_read_timeout 600s;
    proxy_send_timeout 600s;
    proxy_connect_timeout 60s;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

如果已经有域名，例如 `xiaoyinghua.dpdns.org`，把 `server_name _;` 改成：

```nginx
server_name xiaoyinghua.dpdns.org;
```

启用配置：

```bash
sudo ln -sf /etc/nginx/sites-available/sakura-study /etc/nginx/sites-enabled/sakura-study
sudo rm -f /etc/nginx/sites-enabled/default
sudo nginx -t
sudo systemctl reload nginx
```

测试：

```bash
curl http://127.0.0.1/api/health
```

浏览器打开：

```text
http://你的服务器公网IP
```

或：

```text
http://你的域名
```

## 9. 配置域名

这一部分在 Cloudflare 或域名服务商控制台做。

### 9.1 添加 A 记录

添加一条 DNS 记录：

```text
类型：A
名称：你的子域名前缀，根域名可填 @
内容：你的服务器公网IP
代理状态：先选“仅 DNS”
TTL：自动
```

例如：

```text
名称：xiaoyinghua
内容：你的服务器公网IP
```

如果你的完整域名就是 `xiaoyinghua.dpdns.org`，则名称可能填 `xiaoyinghua` 或 `@`，具体看域名所在平台的规则。

### 9.2 为什么 Cloudflare 小云朵有时要关掉

如果开启 Cloudflare 代理，访问路径会变成：

```text
用户 -> Cloudflare -> 你的服务器
```

好处：

- 隐藏源站 IP。
- 有基础缓存和安全能力。

可能的问题：

- 源站 HTTPS 没配好时容易报错。
- 某些临时端口或上传场景不稳定。
- 新手排查问题更困难。

建议顺序：

1. 先用“仅 DNS”跑通。
2. 配好 HTTPS。
3. 确认上传和登录都正常。
4. 再考虑打开代理。

## 10. 配置 HTTPS

有域名后推荐配置 HTTPS。

安装 Certbot：

```bash
sudo apt install -y certbot python3-certbot-nginx
```

申请证书：

```bash
sudo certbot --nginx -d 你的域名
```

例如：

```bash
sudo certbot --nginx -d xiaoyinghua.dpdns.org
```

按提示输入邮箱、同意协议。成功后访问：

```text
https://你的域名
```

检查自动续期：

```bash
sudo certbot renew --dry-run
```

如果 HTTPS 成功，把服务器 `.env` 里的：

```env
APP_PUBLIC_URL=http://你的域名
```

改成：

```env
APP_PUBLIC_URL=https://你的域名
```

然后重启：

```bash
sudo systemctl restart sakura-study
```

```bash
sudo systemctl restart sakura-study
```

然后重启：

```bash

```

## 12. 本地数据迁移到云端

### 12.1 小数据量：网页导出导入

本地打开：

```text
http://127.0.0.1:8000
```

进入数据迁移区域，导出全量备份 ZIP。

云端打开：

```text
https://你的域名
```

进入数据迁移区域，导入刚刚导出的 ZIP。

导入后检查：

- 做题本数量。
- 题目数量。
- 错题数量。
- 教材库。
- AI 老师记忆。
- 历史知识归档。

### 12.2 大数据量：用 scp 上传备份

如果 ZIP 很大，浏览器上传可能慢或者中断，建议用命令上传。

本地 PowerShell：

```powershell
scp .\sakura_backup.zip admin@你的服务器公网IP:/tmp/sakura_backup.zip
```

如果 SSH 有特殊端口：

```powershell
scp -P 端口号 .\sakura_backup.zip admin@你的服务器公网IP:/tmp/sakura_backup.zip
```

服务器上停止服务：

```bash
sudo systemctl stop sakura-study
```

用项目自带恢复逻辑恢复：

```bash
cd /opt/sakura-study
/opt/sakura-study/.venv/bin/python - <<'PY'
from pathlib import Path
from app import restore_backup_zip

zip_path = Path("/tmp/sakura_backup.zip")
result = restore_backup_zip(zip_path.read_bytes())
print(result)
PY
```

重启服务：

```bash
sudo systemctl start sakura-study
sudo systemctl status sakura-study
```

打开网页检查数据是否一致。

## 13. 配置微信提醒和天气推送

Sakura 支持企业微信机器人和 PushPlus。

服务器 `.env` 示例：

```env
APP_PUBLIC_URL=https://你的域名
WEWORK_BOT_WEBHOOK=https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=你的key
PUSHPLUS_TOKEN=
WEATHER_CITY=山东省 日照市 东港区
```

修改后重启：

```bash
sudo systemctl restart sakura-study
```

### 13.1 手动测试推送

可以在网页设置页点击测试按钮。

也可以在服务器命令行测试：

```bash
cd /opt/sakura-study
/opt/sakura-study/.venv/bin/python notify_daily.py --morning
/opt/sakura-study/.venv/bin/python notify_daily.py --night
/opt/sakura-study/.venv/bin/python notify_daily.py --weather
```

如果手动测试能收到，说明 Webhook 或 PushPlus 配置没问题。

### 13.2 配置定时任务

编辑 crontab：

```bash
crontab -e
```

加入：

```cron
0 10 * * * cd /opt/sakura-study && /opt/sakura-study/.venv/bin/python notify_daily.py --morning >> /opt/sakura-study/data/notify_morning.log 2>&1
0 20 * * * cd /opt/sakura-study && /opt/sakura-study/.venv/bin/python notify_daily.py --night >> /opt/sakura-study/data/notify_night.log 2>&1
30 22 * * * cd /opt/sakura-study && /opt/sakura-study/.venv/bin/python notify_daily.py --weather >> /opt/sakura-study/data/notify_weather.log 2>&1
```

查看是否保存成功：

```bash
crontab -l
```

查看日志：

```bash
tail -n 80 /opt/sakura-study/data/notify_weather.log
tail -n 80 /opt/sakura-study/data/notify_morning.log
tail -n 80 /opt/sakura-study/data/notify_night.log
```

### 13.3 注意服务器时区

检查时区：

```bash
timedatectl
```

如果不是中国时间：

```bash
sudo timedatectl set-timezone Asia/Shanghai
timedatectl
```

否则你设置的 22:30 可能不是北京时间 22:30。

```bash
sudo systemctl status sakura-study
curl http://127.0.0.1:8000/api/health
```

### 14.2 ZIP 上传方式

本地重新打包：

```powershell
cd C:\Users\你的用户名\Desktop\demo
Compress-Archive -Path app.py,notify_daily.py,requirements.txt,sakura_*.py,static,docs,deploy,scripts -DestinationPath sakura-code.zip -Force
scp .\sakura-code.zip admin@你的服务器公网IP:/tmp/
```

服务器更新：

```bash
cd /opt/sakura-study
unzip -o /tmp/sakura-code.zip -d /opt/sakura-study
/opt/sakura-study/.venv/bin/pip install -r requirements.txt
sudo systemctl restart sakura-study
```

浏览器看到旧页面时，按：

```text
Ctrl + F5
```

## 15. 常见问题排查

### 15.1 访问不了网页

按顺序检查：

```bash
sudo systemctl status sakura-study
sudo systemctl status nginx
sudo nginx -t
curl http://127.0.0.1:8000/api/health
curl http://127.0.0.1/api/health
```

如果 `127.0.0.1:8000` 正常，但公网打不开：

- 检查云服务商安全组是否开放 80。
- 检查 Nginx 是否启动。
- 检查域名 DNS 是否指向正确 IP。

### 15.2 502 Bad Gateway

通常是 Python 后端没启动。

```bash
sudo systemctl status sakura-study
sudo journalctl -u sakura-study -n 100 --no-pager
```

常见原因：

- `.env` 写错。
- 依赖没安装。
- 项目路径不对。
- systemd 的 `User=` 写错。

### 15.3 上传大 PDF 失败

检查 Nginx 配置是否包含：

```nginx
client_max_body_size 2G;
proxy_read_timeout 600s;
proxy_send_timeout 600s;
```

修改后：

```bash
sudo nginx -t
sudo systemctl reload nginx
```

同时检查磁盘：

```bash
df -h
```

### 15.4 AI 没反应

检查 `.env`：

```bash
cd /opt/sakura-study
grep -E 'LLM_API_KEY|LLM_BASE_URL|LLM_MODEL' .env
```

修改后重启：

```bash
sudo systemctl restart sakura-study
```

如果用中转站，确认：

- `LLM_BASE_URL` 以 `/v1` 结尾。
- 模型名是中转站支持的模型。
- Key 没有复制空格。

### 15.5 定时推送没收到

先手动测试：

```bash
cd /opt/sakura-study
/opt/sakura-study/.venv/bin/python notify_daily.py --weather
```

再检查：

```bash
crontab -l
timedatectl
tail -n 80 /opt/sakura-study/data/notify_weather.log
```

如果手动能收到、定时收不到，多半是：

- crontab 没保存。
- 服务器时区不对。
- 命令路径写错。
- 日志里有 API 或网络错误。

### 15.6 本地和云端数据不一样

最稳的方法：

1. 本地导出全量备份 ZIP。
2. 云端导入全量备份 ZIP。
3. 导入后刷新页面。
4. 对比题库、错题、教材、AI 记忆数量。

不要只复制数据库文件而漏掉 `uploads/` 和 `pages/`，否则题图和 PDF 可能丢失。

### 15.7 服务器磁盘满了

查看磁盘：

```bash
df -h
du -sh /opt/sakura-study/data/*
```

一般占空间最多的是：

```text
data/uploads/
data/pages/
```

处理方式：

- 删除过旧做题本。
- 删除无用导出包。
- 扩容云盘。
- 迁移到更大服务器。
