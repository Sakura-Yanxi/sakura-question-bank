# Sakura 做题集本地到云端部署保姆级教程

这份教程写给第一次部署的小白同学。目标是把本地能跑的 Sakura 做题集，部署到一台 Ubuntu 云服务器上，让手机和电脑都能通过公网访问，并且保留 AI、数据迁移、微信提醒、天气推送等功能。

> 本教程以 Ubuntu 22.04 云服务器为例。阿里云轻量应用服务器、腾讯云轻量服务器、普通云服务器都可以参考。

---

## 0. 先看懂整体思路

Sakura 做题集分成三层：

1. 本地代码
   - `app.py`：后端服务。
   - `static/`：前端页面。
   - `data/`：本地数据库、上传 PDF、拆出来的题图。

2. 云端服务器
   - 放一份同样的代码。
   - 用 Python 在服务器上运行 `app.py`。
   - 用 Nginx 把公网的 80 端口转发到本机的 8000 端口。

3. 数据迁移
   - 本地网页里导出全量 ZIP。
   - 云端网页里导入全量 ZIP。
   - 数据很大时，可以用 SSH 命令上传 ZIP，再在服务器上恢复。

记住一句话：代码用 Git 或 ZIP 传上去，数据用“数据迁移”传上去，密钥写 `.env`，不要上传到 GitHub。

## 1. 本地先跑起来

### 1.1 安装 Python

建议安装 Python 3.11 或 3.12。

Windows 安装时要勾选：

```text
Add Python to PATH
```

安装后打开 PowerShell，检查：

```powershell
python --version
pip --version
```

能看到版本号就可以。

### 1.2 进入项目目录

假设项目在桌面：

```powershell
cd C:\Users\你的用户名\Desktop\demo
```

### 1.3 创建本地虚拟环境

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

如果 PowerShell 不让激活，先执行：

```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

然后重新激活：

```powershell
.\.venv\Scripts\Activate.ps1
```

### 1.4 安装依赖

```powershell
pip install -r requirements.txt
```

### 1.5 启动本地服务

```powershell
python app.py
```

浏览器打开：

```text
http://127.0.0.1:8000
```

看到 Sakura 做题集页面，就说明本地部署成功。

## 2. 准备云服务器

### 2.1 推荐最低配置

小型个人学习 demo：

```text
CPU: 2 核
内存: 2 GB
系统盘: 40 GB 或以上
系统: Ubuntu 22.04
```

如果上传很多 PDF，题图会占空间，建议系统盘至少 40 GB。后期数据变大可以再加数据盘。

### 2.2 云服务器安全组放行端口

在云服务商控制台里，找到“防火墙”或“安全组”，放行：

```text
22   SSH 登录服务器
80   HTTP 网页访问
443  HTTPS 网页访问，可选，后面配置域名证书时用
```

不要直接开放 8000。Sakura 后端只让服务器内部访问，公网交给 Nginx。

### 2.3 SSH 登录服务器

Windows PowerShell 执行：

```powershell
ssh admin@你的服务器公网IP
```

如果服务器用户名是 `root`，就用：

```powershell
ssh root@你的服务器公网IP
```

第一次登录会问 yes/no，输入：

```text
yes
```

## 3. 服务器基础环境安装

以下命令都在服务器 SSH 窗口里执行。

### 3.1 更新系统

```bash
sudo apt update
sudo apt upgrade -y
```

### 3.2 安装必备软件

```bash
sudo apt install -y python3 python3-venv python3-pip nginx git unzip curl fail2ban
```

### 3.3 给 2GB 轻量服务器加一点 swap

如果服务器只有 2GB 内存，上传大 PDF 或恢复大备份时更容易卡。建议加 2GB swap：

```bash
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
free -h
```

看到 Swap 有容量就成功。

## 4. 把代码放到服务器

推荐把项目放到：

```text
/opt/sakura-study
```

### 方式 A：从 GitHub 拉取代码

如果你的代码已经在 GitHub：

```bash
sudo mkdir -p /opt/sakura-study
sudo chown -R $USER:$USER /opt/sakura-study
git clone 你的GitHub仓库地址 /opt/sakura-study
cd /opt/sakura-study
```

如果目录已经存在，以后更新代码用：

```bash
cd /opt/sakura-study
git pull
```

### 方式 B：从本地打包上传代码

如果暂时不用 GitHub，在本地 PowerShell 执行：

```powershell
cd C:\Users\你的用户名\Desktop\demo
Compress-Archive -Path app.py,requirements.txt,notify_daily.py,static -DestinationPath sakura-code.zip -Force
scp .\sakura-code.zip admin@你的服务器公网IP:/tmp/
```

然后在服务器执行：

```bash
sudo mkdir -p /opt/sakura-study
sudo chown -R $USER:$USER /opt/sakura-study
unzip -o /tmp/sakura-code.zip -d /opt/sakura-study
cd /opt/sakura-study
```

## 6. 配置 .env

`.env` 用来保存 API Key、模型地址、推送机器人地址、公网访问地址。不要上传到 GitHub。

在服务器执行：

```bash
cd /opt/sakura-study
nano .env
```

填入下面内容，把占位符改成自己的：

```env
PORT=8000
APP_PUBLIC_URL=http://你的服务器公网IP

LLM_API_KEY=你的DeepSeek或中转站APIKey
LLM_BASE_URL=https://api.deepseek.com
LLM_MODEL=deepseek-chat

WEWORK_BOT_WEBHOOK=你的企业微信群机器人Webhook
PUSHPLUS_TOKEN=

WEATHER_CITY=北京 海淀区
```

保存方式：

```text
Ctrl + O
回车
Ctrl + X
```

保护密钥文件权限：

```bash
chmod 600 /opt/sakura-study/.env
```

说明：

- `APP_PUBLIC_URL` 必须是手机能访问到的地址。没有域名就写 `http://服务器公网IP`。
- `LLM_API_KEY` 是大模型密钥。
- `LLM_BASE_URL` 可以填 DeepSeek 官方地址，也可以填 OpenAI 兼容中转站地址。
- `LLM_MODEL` 填模型名，例如 `deepseek-chat`。
- `WEWORK_BOT_WEBHOOK` 是企业微信群机器人地址。
- `PUSHPLUS_TOKEN` 可以留空。如果同时配置企业微信和 PushPlus，会都尝试推送。

## 7. 配置 systemd 后台服务

这样服务器重启后，Sakura 会自动启动。

```bash
sudo nano /etc/systemd/system/sakura-study.service
```

写入：

```ini
[Unit]
Description=Sakura Study App
After=network.target

[Service]
Type=simple
User=admin
WorkingDirectory=/opt/sakura-study
Environment=PYTHONUNBUFFERED=1
ExecStart=/opt/sakura-study/.venv/bin/python /opt/sakura-study/app.py
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

如果你的服务器用户名不是 `admin`，把这一行改掉：

```ini
User=admin
```

启动服务：

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now sakura-study
sudo systemctl status sakura-study
```

看到 `active (running)` 就成功。

查看日志：

```bash
sudo journalctl -u sakura-study -f
```

退出日志按：

```text
Ctrl + C
```

## 8. 配置 Nginx 反向代理

Sakura 后端监听的是服务器内部的 `127.0.0.1:8000`。Nginx 负责把公网请求转进去。

```bash
sudo nano /etc/nginx/sites-available/sakura-study
```

写入：

```nginx
server {
    listen 80;
    server_name _;

    client_max_body_size 2G;
    proxy_read_timeout 600s;
    proxy_send_timeout 600s;
    proxy_connect_timeout 60s;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header Real-IP $remote_addr;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

启用站点：

```bash
sudo ln -sf /etc/nginx/sites-available/sakura-study /etc/nginx/sites-enabled/sakura-study
sudo rm -f /etc/nginx/sites-enabled/default
sudo nginx -t
sudo systemctl reload nginx
```

浏览器打开：

```text
http://你的服务器公网IP
```

如果能看到 Sakura 做题集，云端部署完成。

## 9. 基础安全设置

### 9.1 开启 UFW 防火墙

```bash
sudo ufw allow OpenSSH
sudo ufw allow 'Nginx Full'
sudo ufw enable
sudo ufw status
```

看到 22、80、443 被允许即可。

### 9.2 开启 fail2ban

```bash
sudo systemctl enable --now fail2ban
sudo systemctl status fail2ban
```

它可以减少 SSH 被暴力尝试登录的风险。

### 9.3 不要做的事

不要把这些内容发到 GitHub：

```text
.env
API Key
企业微信 Webhook
PushPlus token
服务器私钥
```

不要直接开放：

```text
8000
```

后端 8000 只给服务器内部使用。

## 10. 把本地数据迁移到云端

### 方式 A：网页导入，适合普通数据量

本地打开：

```text
http://127.0.0.1:8000
```

进入：

```text
智能每日练习 -> 数据迁移 -> 导出全量 ZIP
```

然后打开云端：

```text
http://你的服务器公网IP
```

进入：

```text
智能每日练习 -> 数据迁移 -> 导入全量 ZIP
```

等待导入完成。数据包含：

- 上传的 PDF。
- 拆出来的题图。
- 题目状态。
- 错题标注。
- 复盘记录。
- AI 记忆。
- 教材库。
- 每日练习规则。

导入时建议同时开着服务器日志：

```bash
sudo journalctl -u sakura-study -f
```

### 方式 B：数据很大时，用 SSH 上传 ZIP

如果备份超过 1GB，浏览器上传可能很慢。可以先用命令传到服务器。

本地 PowerShell：

```powershell
scp .\sakura_backup_你的日期.zip admin@你的服务器公网IP:/tmp/sakura_backup.zip
```

服务器上停止服务，避免恢复时有人写数据：

```bash
sudo systemctl stop sakura-study
cd /opt/sakura-study
```

执行恢复：

```bash
/opt/sakura-study/.venv/bin/python - <<'PY'
from pathlib import Path
from app import restore_backup_zip

zip_path = Path("/tmp/sakura_backup.zip")
result = restore_backup_zip(zip_path.read_bytes())
print(result)
PY
```

恢复后重启：

```bash
sudo systemctl start sakura-study
sudo systemctl status sakura-study
```

浏览器刷新云端页面，看题库数量是否正确。

## 11. 配置每天提醒和天气推送

Sakura 的推送脚本是：

```text
notify_daily.py
```

它会读取 `.env` 里的：

```text
APP_PUBLIC_URL
WEWORK_BOT_WEBHOOK
PUSHPLUS_TOKEN
WEATHER_CITY
```

### 11.1 先手动测试

服务器执行：

```bash
cd /opt/sakura-study
/opt/sakura-study/.venv/bin/python notify_daily.py --morning
/opt/sakura-study/.venv/bin/python notify_daily.py --night
/opt/sakura-study/.venv/bin/python notify_daily.py --weather
```

如果企业微信收到消息，就说明推送配置成功。

### 11.2 设置定时任务

编辑 cron：

```bash
crontab -e
```

加入：

```cron
0 10 * * * cd /opt/sakura-study && /opt/sakura-study/.venv/bin/python notify_daily.py --morning >> /opt/sakura-study/data/notify_morning.log 2>&1
0 20 * * * cd /opt/sakura-study && /opt/sakura-study/.venv/bin/python notify_daily.py --night >> /opt/sakura-study/data/notify_night.log 2>&1
30 22 * * * cd /opt/sakura-study && /opt/sakura-study/.venv/bin/python notify_daily.py --weather >> /opt/sakura-study/data/notify_weather.log 2>&1
```

含义：

- 每天 10:00 发送早间复习提醒。
- 每天 20:00 检查今天是否打卡。
- 每天 22:30 推送明天天气。

查看定时任务：

```bash
crontab -l
```

查看天气推送日志：

```bash
tail -n 50 /opt/sakura-study/data/notify_weather.log
```

## 13. 常见问题

### 13.1 浏览器打不开云端页面

按顺序检查：

```bash
sudo systemctl status sakura-study
sudo systemctl status nginx
sudo nginx -t
curl http://127.0.0.1:8000/api/health
```

如果服务器内部能访问，但公网打不开，检查云平台安全组是否放行 80。

### 13.2 AI 面板提示未配置 API

检查 `.env`：

```bash
cat /opt/sakura-study/.env
```

至少要有：

```env
LLM_API_KEY=你的key
LLM_BASE_URL=https://api.deepseek.com
LLM_MODEL=deepseek-chat
```

修改后重启：

```bash
sudo systemctl restart sakura-study
```

### 13.3 大文件上传卡住

检查 Nginx 配置里有没有：

```nginx
client_max_body_size 2G;
proxy_read_timeout 600s;
proxy_send_timeout 600s;
```

再检查日志：

```bash
sudo journalctl -u sakura-study -f
sudo tail -n 50 /var/log/nginx/error.log
```

如果备份特别大，优先用 SSH 上传 ZIP，再用命令恢复。

### 13.4 微信或企业微信收不到提醒

手动测试：

```bash
cd /opt/sakura-study
/opt/sakura-study/.venv/bin/python notify_daily.py --morning
```

检查 `.env`：

```env
WEWORK_BOT_WEBHOOK=你的企业微信群机器人Webhook
APP_PUBLIC_URL=http://你的服务器公网IP
```

如果 Webhook 写错，企业微信不会收到。

### 13.5 本地数据和云端数据不一样

最稳的做法：

1. 本地导出全量 ZIP。
2. 云端导入全量 ZIP。
3. 导入后刷新页面。
4. 看错题数量、做题本数量、题库数量是否一致。

如果数据很大，用 SSH 迁移方式，不要反复通过浏览器上传。

## 15. 最推荐的最终结构

服务器上建议保持：

```text
/opt/sakura-study/
  app.py
  notify_daily.py
  requirements.txt
  .env
  static/
  data/
    gaoshu_demo.sqlite3
    uploads/
    pages/
```

其中：

- `static/` 是前端。
- `data/` 是所有业务数据。
- `.env` 是密钥配置。
- 更新代码时不要删 `data/`。
- 发给别人代码时不要带 `.env`。

## 16. 一句话版

本地跑通以后，把代码放到服务器 `/opt/sakura-study`，安装依赖，写好 `.env`，用 systemd 保持 `app.py` 后台运行，用 Nginx 把 80 端口转发到 `127.0.0.1:8000`，最后把本地导出的全量 ZIP 导入云端即可。
