# Sakura 做题集

Sakura 做题集是一个面向个人学习与错题复盘的本地优先型智能学习系统。它把 PDF 做题本、模拟卷、教材精读、错题复习、每日练习、AI 学习教练和提醒推送放在同一个学习工作台中，目标是让做过的题不浪费，让薄弱知识点不被遗漏。

> 当前项目仍属于个人学习 demo 和原型系统，不是商业化题库产品。欢迎学习、交流和非商业二次开发，但请尊重原创并保留署名。

---

## 项目定位

Sakura 做题集围绕三个核心问题设计：

1. 做过的题能不能沉淀下来。
2. 错过的题能不能按原因、知识点和复习周期重新出现。
3. AI 能不能基于真实做题证据给出更具体的指导，而不是只生成泛泛的建议。

因此，本项目采用本地优先的思路：PDF 拆页、题图保存、题库筛选、错题状态、统计图表、复习队列和数据迁移都优先在本地完成；只有在需要解析、讲解、变式、反思、教材问答或 AI 教练对话时才调用大模型 API。

## 功能概览

### 资料与题库

- PDF 做题本导入：支持按页拆分题目，保存题图和可提取文本。
- 页码范围导入：可只导入指定起止页，避免整本重复导入。
- 章节识别：优先依据页面右上角章节文本进行知识点和章节归类，并对重复章节名、公众号水印、基础篇等干扰文本做清洗。
- 做题本编辑：资料名称、科目、类型等信息可后续修正，避免填错后必须删除重传。
- 模拟卷模块：模拟卷与普通做题本同级管理，拥有独立上传入口和题库视图。
- 题库筛选：按资料、科目、知识点、章节、掌握状态、关键词等维度筛选。
- 题图查看：支持题图放大、拖动、双击查看，方便检查 PDF 拆页后的内容。
- 题目裁剪：支持手动微调题目边界，缓解一页多题或题号与选项跨页的问题。
- 空资料清理：当某个资料下题目被清空后，可同步清理无效资料记录。

### 错题与复习

- 状态记录：未做、做对、做错、半会、需复习等状态。
- 元认知错因标签：
  - 计算失误
  - 公式遗忘
  - 逻辑死角
  - 题意理解偏差
- 复习调度：做错后进入复习队列，后续做对会逐步拉长复习间隔，参考 1 / 3 / 7 / 14 / 30 天的间隔复习思路。
- 每日练习：默认从当前仍有价值的错题、半会题、到期复习题和历史薄弱题中生成，不是无脑从原题库抽题。
- 自定义练习规则：可按科目、资料、知识点、章节设置每日推送范围和数量。
- 知识溯源：当某章节正确率过低时，可在每日练习中穿插前置基础题，帮助回补根因知识。
- 错题导出：可按资料、科目、知识点、章节筛选导出 PDF，支持全选和自定义选择。
- 统计画像：包含知识点分布、章节正确率、错因统计和错因雷达图。

### AI 学习教练

AI 学习教练不是单纯的聊天框，而是结合学习档案、错题证据、复习队列和老师记忆的 tutor agent 雏形。

当前支持：

- API 可用性测试。
- 错题启发式提示：
  - Level 1：只给核心概念或定理。
  - Level 2：给第一步关键变形或切入点。
  - Level 3：生成完整 LaTeX 解析。
- 举一反三变式：
  - Base：换数不换逻辑。
  - Advanced：改变求解目标。
  - Pro：跨章节综合。
- AI 老师对话：可围绕学习计划、错题复盘、知识点讲解和节奏调整连续追问。
- 老师记忆：可将重要对话压缩导入记忆，用于后续个性化指导。
- 学习档案：基于做题记录、薄弱知识点、错因结构和复习队列生成学习画像。
- 历史知识归档：可查看历史档案和 AI 记忆，并支持清理不需要的建议记录。
- 外部经验库：可以导入他人的备考经验，但系统会区分“外部经验”和“个人真实证据”。

### 教材精读

- 教材 PDF 上传：按页保存页面截图并提取段落。
- 页码和段落定位：可围绕指定页码、指定段落向 AI 提问。
- 连续追问：支持对同一教材内容进行多轮解释。
- 记忆压缩：可将高价值解释压缩导入老师记忆，后续复习时继续使用。
- 公式渲染：前端集成 MathJax，尽量保证 AI 生成的 LaTeX 公式正常显示。

### 总结与反思

- 周复盘：按科目统计本周做题量、正确量、错误量、薄弱章节和错因。
- 月复盘：拉长周期观察知识点稳定性和复习完成情况。
- AI 反思：可调用大模型指出当前不足、阶段重点和后续规划。
- 数据口径：复盘优先依据真实录入和标注的做题记录，不把不存在的练习硬算进去。

### 提醒与推送

- 企业微信机器人推送。
- PushPlus 推送。
- 早晚提醒时间可自定义。
- 天气推送可配置地区和时间。
- 打卡方式支持本地模式和云端链接模式。
- 推送测试按钮用于检查配置是否可用。

### 数据迁移

- 支持本地数据导出。
- 支持从备份恢复。
- 迁移目标包括题库记录、错题状态、标注信息、学习档案、老师记忆、资料元数据和必要的本地文件索引。
- `.env`、API Key、日志和临时压缩包不应进入 Git 仓库。

## 技术栈

- 后端：Python 标准库 `http.server`
- 数据库：SQLite
- PDF 处理：PyMuPDF
- 图像处理：Pillow
- 前端：原生 HTML / CSS / JavaScript
- 数学公式：MathJax
- 图标：Lucide
- AI 调用：OpenAI SDK，支持 DeepSeek 和 OpenAI Compatible API
- 推送：企业微信机器人、PushPlus

```text
.
├── app.py                         # 后端入口与核心 HTTP 处理

│   ├── app.js                     # 前端交互与 API 调用
│   └── styles.css                 # 前端样式
├── sakura_*.py                    # 按领域拆分的后端模块
├── tests/
│   └── smoke_refactor.py          # 关键重构冒烟测试
├── docs/
│   ├── architecture.md
│   └── 本地到云端部署保姆级教程.md
├── deploy/
│   ├── azure-vm.md
│   ├── local-to-cloud.md
│   └── novm-mini.md
└── data/                          # 本地数据目录，默认不提交
```

后端已经按功能逐步拆分为多个 `sakura_*.py` 模块，例如：

- `sakura_pdf.py`：PDF 渲染、题目切片、页码范围处理。
- `sakura_classify.py`：章节识别、章节清洗、导入分类。
- `sakura_questions.py`：题目增删改查、章节统计、OCR 文本追加。
- `sakura_documents.py`：资料和教材记录管理。
- `sakura_teacher_memory.py`：AI 老师记忆、经验库和对话回合记录。
- `sakura_backup.py`：备份导出、恢复和迁移参数解析。
- `sakura_filters.py`：题库筛选和搜索条件构建。
- `sakura_routes.py`：API 路由注册。

## 快速开始

### 1. 安装依赖

```powershell
pip install -r requirements.txt
```

### 2. 配置环境变量

复制 `.env.example` 为 `.env`，然后按需填写：

```powershell
copy .env.example .env
```

最低限度可以不配置 AI Key，系统仍能进行 PDF 导入、题库管理、错题标注、统计和本地复习；但 AI 解析、AI 教练、教材问答、反思总结等能力会降级或不可用。

### 3. 启动服务

```powershell
python app.py
```

### 4. 打开浏览器

```text
http://127.0.0.1:8000
```

## 常用环境变量

```env
PORT=8000
APP_PUBLIC_URL=http://127.0.0.1:8000

SAKURA_ADMIN_PASSWORD=change-me
SAKURA_AUTH_SECRET=replace-with-a-long-random-string
SAKURA_DEMO_MODE=0

LLM_API_KEY=
LLM_BASE_URL=https://api.deepseek.com/v1
LLM_MODEL=deepseek-chat

WEWORK_BOT_WEBHOOK=
PUSHPLUS_TOKEN=

REMIND_MORNING_ON=0
REMIND_MORNING_TIME=10:00
REMIND_NIGHT_ON=0
REMIND_NIGHT_TIME=20:00
REMIND_WEATHER_ON=0
REMIND_WEATHER_TIME=22:30
WEATHER_CITY=北京
```

说明：

- `SAKURA_ADMIN_PASSWORD`：公网部署时建议开启，用于登录保护。
- `SAKURA_AUTH_SECRET`：登录签名密钥，应使用足够长的随机字符串。
- `SAKURA_DEMO_MODE=1`：演示模式，写入类接口会被限制，适合给朋友体验。
- `LLM_API_KEY`：大模型 API Key。
- `LLM_BASE_URL`：OpenAI Compatible API 地址，例如 DeepSeek 或中转站地址。
- `LLM_MODEL`：模型名。
- `APP_PUBLIC_URL`：公网访问地址，用于生成手机端回填链接和推送链接。
- `WEWORK_BOT_WEBHOOK`：企业微信群机器人 Webhook。
- `PUSHPLUS_TOKEN`：PushPlus Token。

兼容历史配置时，模型密钥读取优先级为：

```text
LLM_API_KEY > MIMO_API_KEY > DEEPSEEK_API_KEY
```

## Token 与费用说明

不会消耗大模型 token 的操作：

- PDF 上传和拆页。
- 本地题图保存。
- 基础文本提取。
- 题库筛选。
- 错题状态记录。
- 知识点和章节统计。
- 本地复习队列计算。
- 数据导入导出。
- 天气查询本身。

会消耗大模型 token 的操作：

- AI 错题分析。
- Full Solution 完整解析。
- 举一反三变式生成。
- AI 学习教练对话。
- 教材精读问答。
- 周期反思复盘。
- 学习档案深度总结。
- 老师记忆压缩。

设计原则是：能用本地规则解决的事情尽量本地完成，只有真正需要语言理解、讲解和规划时才调用模型。

## 部署说明

项目可以本地运行，也可以部署到轻量云服务器、Azure 学生服务器或临时演示服务器。

已有部署文档：

- `docs/本地到云端部署保姆级教程.md`：给小白用户看的完整部署教程。
- `deploy/local-to-cloud.md`：本地到云端迁移说明。
- `deploy/azure-vm.md`：Azure VM 部署参考。
- `deploy/novm-mini.md`：轻量演示服务器部署参考。

公网部署建议：

1. 配置 `SAKURA_ADMIN_PASSWORD` 和 `SAKURA_AUTH_SECRET`。
2. 不要把 `.env`、数据库、上传 PDF、日志、导出压缩包提交到 GitHub。
3. 演示环境建议开启 `SAKURA_DEMO_MODE=1`。
4. 使用域名时，将 DNS 指向服务器公网 IP，并确认服务商防火墙开放 80 / 443。
5. 如果部署在中国大陆服务器，域名访问通常需要考虑备案要求。

## 数据与安全

本项目默认不提交以下内容：

- `.env`
- `data/`
- SQLite 数据库
- 上传的 PDF
- 拆分后的题图
- 日志文件
- 导出的 zip 或 PDF
- 部署密钥

请不要把自己的 API Key、企业微信 Webhook、PushPlus Token、服务器密码等敏感信息写入 README、代码或公开 issue。

用户自行上传的 PDF、教材、题库等资料由使用者自行负责版权合规。本项目不提供任何盗版教材、盗版题库或商业资料。

## 当前状态

当前版本已经形成较完整的个人学习闭环：

```text
PDF / 教材导入
  -> 题目拆分与章节识别
  -> 做题状态记录
  -> 错因标注
  -> 间隔复习
  -> 每日练习
  -> AI 解析与变式
  -> 周期复盘
  -> 学习档案更新
  -> AI 老师策略调整
```

仍可继续增强的方向：

- 更稳定的模拟卷自动分题。
- 更接近 FSRS 的复习调度算法。
- 多用户账户体系。
- 云端同步和权限隔离。
- 更完整的移动端错题回填体验。
- 更规范的开源许可证文件。
- 更强的 AI 老师评估与自我校准机制。
